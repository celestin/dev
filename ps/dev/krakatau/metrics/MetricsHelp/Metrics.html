Essential Metrics for C/C++

Method

eV(G) - Essential Complexity

Essential Complexity (ev(G)) is a measure of the degree to which this
method contains unstructured constructs. This metric measures the
degree of structuredness and the quality of the code. It is used to
predict the maintenance effort and to help in the modularization process.

V(G) - Cyclomatic Complexity

Cyclomatic Complexity (v(G)) is a measure of the complexity of the method's decision
structure. It is the number of linearly independent paths and therefore, the minimum
number of paths that should be tested.

A v(G) value greater than 7 may indicate an overly complex method.

V'(G) - Extended Cyclomatic Complexity

V'(G) = OC / V(G)

V'(G) provides a complexity ratio, based the operational complexity and cyclomatic complexity of a method.

File

D - Halstead program Difficulty

A measure of how difficult this file's code is to understand.

E - Halstead program Effort

Effort = Difficulty * Volume for this file.

N - Halstead program Length

Halstead program Length (N) for this file is calculated as N1 + N2.

N1 = Total number of Operators
N2 = Total number of Operands

n - Halstead program Vocabulary

Halstead program Vocabulary (n) for this file is calculated as n1 + n2.

n1 = Number unique or distinct Operators
n2 = Number unique or distinct Operands

n1 - Halstead number of unique operators

The number of unique or distinct operators in the file.

n1 is part of the Halstead set, and of little interest on its own. When combined with the other Halstead fundamental metrics, it can be used to determine Volume, Difficult, Effort and Cyclomatic complexity.

N1 - Halstead total number of operators

The total number of operators in the file.

N1 is part of the Halstead set, and of little interest on its own. When combined with the other Halstead fundamental metrics, it can be used to determine Volume, Difficult, Effort and Cyclomatic complexity.

N2 - Halstead total number of operands

The total number of operands in the file.

N2 is part of the Halstead set, and of little interest on its own. When combined with the other Halstead fundamental metrics, it can be used to determine Volume, Difficult, Effort and Cyclomatic complexity.

The number of unique or distinct operands in the file.

n2 is part of the Halstead set, and of little interest on its own. When combined with the other Halstead fundamental metrics, it can be used to determine Volume, Difficult, Effort and Cyclomatic complexity.

Project

AVPATHS - Average Depth of Paths

AVPATHS is calculated by counting the number and size of all paths from all methods,
and then dividing that number by the number of methods which had other method calls.

In other words, the average depth of paths from methods that have path at all.

The total number of methods in the project.

A count of the total number of semicolons in the project excluding those within comments

SLOC - Source lines of code

Number of Source lines in the project, excluding whitespace and comments.

This is not a count of semicolons or distinct statements, but a count of
physical lines that contain source code.

If you are interested in "logical" lines, see NSC.

&lt;&lt; " Hello"

The ratio, at project level, of comment lines to source lines of code.

WLOC - Whitespace lines of code

Whitespace lines of code is the number of lines in the project containing solely whitespace - the difference between total lines and source+comment lines.

ACLOC - Average Lines per Class

This metric gives the average Class size in terms of LOC.

AHF - Attribute Hiding Factor

AttributesVisible is calculated by summing the visibility (i.e. private=1, public=0,
protected=InheritenceTreeSize/NumClasses) of each attribute in respect to the other
classes in the project.

AIF - Attribute Inheritence Factor

Inherited Attributes is the sum of NAIC for each class in the project.

AMLOC - Average Lines per Method

This metric gives the average Method size in terms of LOC.

HIER - Hierarchical Depth

Hierarchical Depth

MHF - Method Hiding Factor

MethodsVisible is calculated by summing the visibility (i.e. private=1, public=0, protected=InheritenceTreeSize/NumClasses) of each method in respect to the other classes in the project.

MIF - Method Inheritence Factor

The ratio of inherited methods to the total number of methods.

InheritedMethods is a sum of NOIC (No. of Operations Inherited) for each class.

Polymorphism Factor can be used to measure the extent to which classes within the project are polymorphic.

Class

CBO - Coupling Between Object classes

Calculated by totalling the number of unique types of attribute within each class. The count only includes user-defined object types. The larger this number, the greater coupling between object classes.

CSA - Class Size (Attributes)

The number of attributes of a class.

CSAO - Class Size (Attributes & Operations)

The total number of attributes and operations for a class.

CSI - Class Specialisation Index

(NOOC * DIT) / TotalMethods

NOOC = No. of Operations overridden.
DIT = Depth of Inheritance Tree - From the current class, traces up entire inheritence heirarchy counting which route is the longest.

CSO - Class Size (Operations)

The number of operations for a class.

DIT - Depth of Inheritence Tree

From the current class, traces up entire inheritence heirarchy counting which route is the longest. The value for DIT is stored against the current class, so for the root class DIT=1, first-level subclasses DIT=2, and so on.

LCOM - Lack of Cohesion Methods

This metric quantifies how much intercourse the methods of a class
have with the member variables of that class.

LOCM is calculated by building a list of member variables and a
count of the number of references to each variable (usage) in
all methods of this class.

Then, a sum of the ratios of usage / TotalMethods is calculated.

Finally, LOCM is returned as this SumOfRatios / TotalAttributes.

NAAC - Number of Attributes Added

NAAC is a count of the number of new Attributes that a class has in respect to its immediate super-class. For a root class, NAAC == CSA.

NAIC - Number of Attributes Inherited

NAIC is a count of the number of Attributes that a class has inherited from its super-classes. For a root class, NAIC == 0.

NAOC - Number of Operations Added

NOAC is a count of the number of new Operations that a class has in respect to its immediate super-class. For a root class, NAOC == CSO.

NOIC - Number of Operations Inherited

NOIC is a count of the number of Operations that a class has inherited from its super-classes. For a root class, NOIC == 0.

NOOC - Number of Operations Overridden

NOOC is a count of the number of Operations that a class has overridden. For a root class, NOOC == CSO.

NPavgC - Average Number of Method Parameters

NPavgC = TotalNP / NumMethods.

The average number of parameters per method. This metric, if over 10, would indicate that too much information is being passed into methods, and that more attributes should be added to the class.

NSUB - Number of Sub Classes

NSUB is a count of the number of immediate sub-classes of a class.

Osavg - Average Operation Size

For a class, Osavg = WMC / NumMethods

OSavg is a measure of the complexity of a class. Classes with OSavg greater than 10 should perhaps be redesigned.

PA - Private Attribute Usage

PA is the sum of all references to distinct private attributes within all methods of a class.

As a guideline, if this number is lower than NumMethods, perhaps parameters and local variables are being overused, and the class methods could be work more closely with the class attributes.

PPPC - Percentage Public/Protected members

PPPC is simply the percentage of public and protected members of a class in respective to all members of the class. I.e.:

PPPC = PublicProtectedMembers / CSAO.

RFC - Response For Class

RFC is the number of methods in a class, plus the number of distinct methods called by those methods.

WMC - Weighted Methods in Class

WMC is the sum of the Cyclomatic complexity for all methods in the class. This number reflects the complexity of the class, and will be affected not only by the V(g) of each method but also the number of methods.

Classes with WMC larger than 100 should perhaps be redesigned.

MethodsVisible is calculated by summing the visibility (i.e. private=1, public=0, protected=InheritenceTreeSize/NumClasses) of each method in respect to the other classes in the project.

MIF - Method Inheritence Factor

The ratio of inherited methods to the total number of methods.

InheritedMethods is a sum of NOIC (No. of Operations Inherited) for each class.

NCLASS - Number of Classes

The total number of classes in the project.

NORC - Number of Root Classes

NORC is a count of the number root classes for a project.

POF - Polymorphism Factor

Polymorphism Factor can be used to measure the extent to which classes within the project are polymorphic.

File

NTC - Number of Top-level Classes

A count of the number of Top-level Classes in the file.

NTI - Number of Top-level Interfaces

A count of the number of Top-level Interfaces in the file.

Class

CBO - Coupling Between Object classes

Calculated by totalling the number of unique types of attribute within each class. The count only includes user-defined object types. The larger this number, the greater coupling between object classes.

CSA - Class Size (Attributes)

The number of attributes of a class.

CSAO - Class Size (Attributes & Operations)

The total number of attributes and operations for a class.

CSI - Class Specialisation Index

(NOOC * DIT) / TotalMethods

NOOC = No. of Operations overridden.
DIT = Depth of Inheritance Tree - From the current class, traces up entire inheritence heirarchy counting which route is the longest.

CSO - Class Size (Operations)

The number of operations for a class.

DIT - Depth of Inheritence Tree

From the current class, traces up entire inheritence heirarchy counting which route is the longest. The value for DIT is stored against the current class, so for the root class DIT=1, first-level subclasses DIT=2, and so on.

LCOM - Lack of Cohesion Methods

This metric quantifies how much intercourse the methods of a class
have with the member variables of that class.

LOCM is calculated by building a list of member variables and a
count of the number of references to each variable (usage) in
all methods of this class.

Then, a sum of the ratios of usage / TotalMethods is calculated.

Finally, LOCM is returned as this SumOfRatios / TotalAttributes.

NAAC - Number of Attributes Added

NAAC is a count of the number of new Attributes that a class has in respect to its immediate super-class. For a root class, NAAC == CSA.

NAIC - Number of Attributes Inherited

NAIC is a count of the number of Attributes that a class has inherited from its super-classes. For a root class, NAIC == 0.

NAOC - Number of Operations Added

NOAC is a count of the number of new Operations that a class has in respect to its immediate super-class. For a root class, NAOC == CSO.

NOIC - Number of Operations Inherited

NOIC is a count of the number of Operations that a class has inherited from its super-classes. For a root class, NOIC == 0.

NOOC - Number of Operations Overridden

NOOC is a count of the number of Operations that a class has overridden. For a root class, NOOC == CSO.

NPavgC - Average Number of Method Parameters

NPavgC = TotalNP / NumMethods.

The average number of parameters per method. This metric, if over 10, would indicate that too much information is being passed into methods, and that more attributes should be added to the class.

NSUB - Number of Sub Classes

NSUB is a count of the number of immediate sub-classes of a class.

Osavg - Average Operation Size

For a class, Osavg = WMC / NumMethods

OSavg is a measure of the complexity of a class. Classes with OSavg greater than 10 should perhaps be redesigned.

PA - Private Attribute Usage

PA is the sum of all references to distinct private attributes within all methods of a class.

As a guideline, if this number is lower than NumMethods, perhaps parameters and local variables are being overused, and the class methods could be work more closely with the class attributes.

PPPC - Percentage Public/Protected members

PPPC is simply the percentage of public and protected members of a class in respective to all members of the class. I.e.:

PPPC = PublicProtectedMembers / CSAO.

RFC - Response For Class

RFC is the number of methods in a class, plus the number of distinct methods called by those methods.

WMC - Weighted Methods in Class

WMC is the sum of the Cyclomatic complexity for all methods in the class. This number reflects the complexity of the class, and will be affected not only by the V(g) of each method but also the number of methods.

Classes with WMC larger than 100 should perhaps be redesigned.

Interface

ISA - Interface Size (Attributes)

The number of attributes of an interface.

ISAO - Interface Size (Attributes & Operations)

The total number of attributes and operations for an interface.

ISO - Interface Size (Operations)

The number of operations for an interface.

NOCI - Number of Child Interfaces

NOCC is a count of the number of immediate sub-interfaces of an interface.

PPPC - Percentage Public/Protected members

PPPC is simply the percentage of public and protected members of an interface in respective to all members of the interface. I.e.:

PPPC = PublicProtectedMembers / CSAO.

For a class, Osavg = WMC / NumMethods

OSavg is a measure of the complexity of a class. Classes with OSavg greater than 10 should perhaps be redesigned.

PA - Private Attribute Usage

PA is the sum of all references to distinct private attributes within all methods of a class.

As a guideline, if this number is lower than NumMethods, perhaps parameters and local variables are being overused, and the class methods could be work more closely with the class attributes.

PPPC - Percentage Public/Protected members

PPPC is simply the percentage of public and protected members of a class in respective to all members of the class. I.e.:

PPPC = PublicProtectedMembers / CSAO.

RFC - Response For Class

RFC is the number of methods in a class, plus the number of distinct methods called by those methods.

SLOC - Source lines of code

Number of Source lines in this class, excluding whitespace and comments.

This is not a count of semicolons or distinct statements, but a count of

&lt;&lt; " Hello"

WMC - Weighted Methods in Class

WMC is the sum of the Cyclomatic complexity for all methods in the class. This number reflects the complexity of the class, and will be affected not only by the V(g) of each method but also the number of methods.

Classes with WMC larger than 100 should perhaps be redesigned.

N2 = Total number of Operands

n2 = Number unique or distinct Operands

Method

CONTROL - Number of control statements

A count of the number of control statements in the method.

Every occurance of a control keyword (such as if, else, while,
switch, case, etc.) causes this count to be incremented by 1.

eV(G) - Essential Complexity

Essential Complexity (ev(G)) is a measure of the degree to which this
method contains unstructured constructs. This metric measures the
degree of structuredness and the quality of the code. It is used to
predict the maintenance effort and to help in the modularization process.

EXEC - Number of executable statements

A count of the number of executable statements in the method.

Every statement that is not a control statement (see CONTROL) causes
this count to be incremented by 1.

LOC - Lines of code

Number of Lines in this method, including source, whitespace and comments.

Effectively, a count of the number of newlines in the method.

SLOC - Source lines of code

Number of Source lines in this method, excluding whitespace and comments.

This is not a count of semicolons or distinct statements, but a count of
physical lines that contain source code.

int i=0; float j=0; // This is 1 SLOC

cout &lt;&lt; "Testing"
&lt;&lt; " Hello"
&lt;&lt; " there." ; // This is 3 SLOCs

V(G) - Cyclomatic Complexity

Cyclomatic Complexity (v(G)) is a measure of the complexity of the method's decision
structure. It is the number of linearly independent paths and therefore, the minimum
number of paths that should be tested.

A v(G) value greater than 7 may indicate an overly complex method.

C_COM - C style comments

The number of C style comments in the project.

A C style comment begins /* and ends */.

In the following example C_COM = 1.

int i = 0 ; /* This is a useful
variable here for
many purposes */

COM_LOC - Comment lines of code

The total number of lines of comment in the project.

COM_RAT - Comments Ratio (Project)

The ratio of comment lines to lines of code.

CPP_COM - C++ style comments

The number of C++ style comment lines in the project.

A C++ style comment begins // until the end of the line.

In the following example CPP_COM=2.

int i = 0 ; // Index variable
float pi = 3.14 ; // Handy pi variable

NCLASS - Number of Classes

The total number of classes in the project.

NFILE - Number of Files

The total number of files in the project.

NMETH - Number of Methods

The total number of methods in the project.

This count includes class methods and C functions.

NSC - Number of Semicolons

A count of the total number of semicolons in the project excluding those within comments
and string literals. This is useful for approximating "logical lines of code". E.g.:

cout &lt;&lt; "Hello" &lt;&lt; endl ; /* output ; return */
cout &lt;&lt; "Hello ; World" &lt;&lt; endl ;

NSC = 2.

SLOC - Source lines of code

Number of Source lines in the project, excluding whitespace and comments.

This is not a count of semicolons or distinct statements, but a count of
physical lines that contain source code.

If you are interested in "logical" lines, see NSC.

int i=0; float j=0; // This is 1 SLOC

cout &lt;&lt; "Testing"
&lt;&lt; " Hello"
&lt;&lt; " there." ; // This is 3 SLOCs

TCOM_RAT - True Comments Ratio (Project)

The ratio, at project level, of comment lines to source lines of code.

WLOC - Whitespace lines of code

Whitespace lines of code is the number of lines in the project containing solely whitespace - the difference between total lines and source+comment lines.
