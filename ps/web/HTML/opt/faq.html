    • What are the platforms/VMs supported by Optimizeit? 

    Optimizeit supports the following platforms:

    • Windows NT/95/98/2000
    • Sparc/UltraSparc Solaris
    • Solaris Intel
    • Linux i386 Certified on RedHat

    Optimizeit supports all VMs  based on Sun's VM and the IBM VM.

    What type of Java programs can be profiled by Optimizeit? / Can I profile servlets with OptimizeIt? / Can I profile EJBs with OptimizeIt? 

    Optimizeit can profile any Java code: application, applets, servlets, EJBs, JSPs, etc.
    Profiling EJBs or JSPs can be done by profiling the Application Server that runs your Java code. OptimizeIt integrates with all Application/Web Servers.

    If the source code is available, Optimizeit can pinpoint the exact line of code using the source code viewer.

    Can I profile an application running in an Application/Web Server with OptimizeIt? 

    OptimizeIt integrates with the following Application/Web Servers:

    • ATG Dynamo
    • iPlanet Web Server Enterprise Edition
    • Apache JServ
    • Allaire Jrun
    • Sun ServletRunner and JavaWebServer
    • Jakarta Tomcat
    • Unify ServletExec
    • Apple Computer WebObjects
    • BEA WebLogic
    • IBM Websphere

    OptimizeIt comes with wizards that do the integration for you. The wizards are available from the menu Tool/Application server integration.

    OptimizeIt  also provides tutorials that explain the integration. Tutorials are accessible from the menu Info/Tutorials.

    What are the IDEs Optimizeit integrate with? 

    Optimizeit integrates with the following IDEs:

    • IBM VisualAge
    • Inprise JBuilder
    • WebGain VisualCafe
    • Oracle JDeveloper

    Can I start Optimizeit from my code? / Does Optimizeit comes with an open API? 

    Optimizeit provides open APIs allowing you to start/pause/resume CPU or memory profiling directly from your source code. This is especially useful when you want to profile very precise commands of your Java programs.

    Here is an example that starts the Optimizeit audit system:

    /* Start the audit system, waiting for the OptimizeIt UI to attach */
    System.out.println("Waiting for OptimizeIt...");
    Audit.start(1470,Audit.PROFILERS_ALWAYS_ENABLED|Audit.WAIT_FOR_OPTIMIZEIT);
    System.out.println("Attach done.");

    More information about the open APIs can be found in the OptimizeIt's user manual: menu Info/Help, Chapter 3 Profiling a Java Program

    Can I profile a program running on a different machine? / Can I remotely profile a server running a different OS than my local machine? 

    Optimizeit allows you to profile Java program running on a different machine. It is similar to testing a Java program started from the command line.

    More information about remote profiling can be found in the OptimizeIt's user manual: menu Info/Help, Chapter 3 Profiling a Java Program.

    Optimizeit allows you to profile any Java program on a server running a different OS than your local machine. The process is the same as the remote profiling described above.

    Can I take snapshots of an application? 

    OptimizeIt allows you to start the profiling session from the command line and automatically generate snapshots at fixed intervals. Snapshots can be reloaded in OptimizeIt for later analysis. This feature can be used to profile an application server over a long period or in a production environment for example.

    Offline profiling also limits overhead since you do not attach to the application.
